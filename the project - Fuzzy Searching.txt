## The Three Algorithms — What They Do & When to Use Each

### Algorithm 1: `token_set_ratio` — handles word order & partial overlap

```python
from rapidfuzz import fuzz

# Sorts both strings by word alphabetically, then compares
# Makes word ORDER irrelevant

print(fuzz.token_set_ratio("4 part grinder", "GRINDER 4 PART"))  # → 100
print(fuzz.token_set_ratio("red apple juice", "apple juice red")) # → 100
print(fuzz.token_set_ratio("hookah small", "small china hookah")) # → 100

# Also handles subset matching — if query is a subset of product name:
print(fuzz.token_set_ratio("hookah", "china hookah small"))       # → 100
print(fuzz.token_set_ratio("glass pipe", "10 inch glass pipe beaker")) # → 100
```

**Use when:** product names have inconsistent word order, or user types partial names.

---

### Algorithm 2: `WRatio` — handles typos & character-level errors

```python
from rapidfuzz import fuzz

# Smart combination internally — best for typo tolerance
print(fuzz.WRatio("hooka", "hookah"))        # → 91  (missing h)
print(fuzz.WRatio("grider", "grinder"))      # → 91  (missing n)
print(fuzz.WRatio("juul", "JUUL"))           # → 100 (case insensitive)
print(fuzz.WRatio("marlbro", "marlboro"))    # → 92  (transposition)
print(fuzz.WRatio("cigartte", "cigarette"))  # → 94  (missing e)
```

**Use when:** users make spelling mistakes (which they always do).

---

### Algorithm 3: `partial_ratio` — catches short query inside long string

```python
from rapidfuzz import fuzz

# Finds the best matching substring window
print(fuzz.partial_ratio("glass", "10 INCH AQUA CLEAR GLASS BEAKER 9MM")) # → 100
print(fuzz.partial_ratio("energy", "5 HOUR EXTRA ENERGY BERRY BOX"))       # → 100
print(fuzz.partial_ratio("blue", "ACID KRUSH BLUE TINS UNIT OF 5"))        # → 100
print(fuzz.partial_ratio("usb", "USB-C FAST CHARGER 65W"))                 # → 100
```

**Use when:** users type one keyword from a long product name.

## Complete Standalone Fuzzy Search Class

Copy this entire block into any project:

```python
"""
fuzzy_search.py
---------------
Self-contained fuzzy search using the 3-algorithm blend.
No external dependencies except: pip install rapidfuzz

Usage:
    from fuzzy_search import FuzzySearchEngine
    
    engine = FuzzySearchEngine(products)
    results = engine.search("hooka grider", top_k=10)
"""

from rapidfuzz import fuzz, process
import re
from typing import List, Dict, Any


# ── Text normalizer ────────────────────────────────────────────────────────────

def normalize(text: str) -> str:
    """
    Clean text for matching:
    - lowercase
    - remove prices like $20.00
    - remove content in brackets like (9MM) or [BOX]
    - remove special characters
    - collapse whitespace
    """
    if not text or not isinstance(text, str):
        return ""
    text = text.lower()
    text = re.sub(r'\$\s*\d+\.?\d*', ' ', text)       # remove prices
    text = re.sub(r'[\[\(\{].*?[\]\)\}]', ' ', text)  # remove brackets
    text = re.sub(r'[^a-z0-9\s]', ' ', text)          # keep only alphanum
    text = re.sub(r'\s+', ' ', text).strip()
    return text


# ── Core scoring function ──────────────────────────────────────────────────────

def blend_score(query: str, normalized_text: str, raw_text: str) -> float:
    """
    Three-algorithm blend scored against both normalized and raw text.
    Takes the best of the two.

    Weights:
        token_set_ratio  0.5  — word order, partial overlap
        WRatio           0.3  — typo tolerance
        partial_ratio    0.2  — short query inside long string
    """
    # Score against normalized text
    ts_n = fuzz.token_set_ratio(query, normalized_text)
    wr_n = fuzz.WRatio(query, normalized_text)
    pr_n = fuzz.partial_ratio(query, normalized_text)
    score_normalized = 0.5 * ts_n + 0.3 * wr_n + 0.2 * pr_n

    # Score against raw text (preserves numbers, original casing)
    ts_r = fuzz.token_set_ratio(query, raw_text)
    wr_r = fuzz.WRatio(query, raw_text)
    pr_r = fuzz.partial_ratio(query, raw_text)
    score_raw = 0.5 * ts_r + 0.3 * wr_r + 0.2 * pr_r

    return max(score_normalized, score_raw)


# ── Main engine ────────────────────────────────────────────────────────────────

class FuzzySearchEngine:
    """
    Fuzzy search engine using a 3-algorithm blend.

    Parameters
    ----------
    items : list of dict
        Each dict must have at least one text field.
        Example: [{"id": 1, "name": "Hookah Small", "brand": "China", "category": "Hookahs"}]

    text_fields : list of str
        Fields to combine into the searchable text.
        Default: ["name", "brand", "category"]

    min_score : float
        Minimum score (0-100) to include in results. Default: 35.
    """

    def __init__(
        self,
        items: List[Dict[str, Any]],
        text_fields: List[str] = None,
        min_score: float = 35.0,
    ):
        self.items = items
        self.text_fields = text_fields or ["name", "brand", "category"]
        self.min_score = min_score
        self._build_index()

    def _build_index(self):
        """Pre-compute raw and normalized searchable strings for every item."""
        self.raw_strings = []
        self.normalized_strings = []

        for item in self.items:
            # Combine all text fields into one string
            parts = [str(item.get(f, '')) for f in self.text_fields]
            raw = ' '.join(p for p in parts if p and p.lower() not in ('nan', 'none', ''))
            self.raw_strings.append(raw)
            self.normalized_strings.append(normalize(raw))

    def search(self, query: str, top_k: int = 20) -> List[Dict[str, Any]]:
        """
        Search for items matching the query.

        Returns list of dicts, each original item enriched with:
            score       — final blend score (0-100)
            score_pct   — same as score (for display as %)
        Sorted by score descending.
        """
        query = normalize(query)
        if not query:
            return []

        # ── Pass 1: Fast WRatio scan to get top candidates ────────────────────
        # Fetch 2x top_k so the re-scoring pass has enough to work with
        fast_matches = process.extract(
            query,
            self.normalized_strings,
            scorer=fuzz.WRatio,
            limit=top_k * 2,
        )

        # ── Pass 2: Re-score with full 3-way blend ────────────────────────────
        results = []
        seen = set()

        for _text, _fast_score, index in fast_matches:
            if index in seen:
                continue
            seen.add(index)

            score = blend_score(
                query,
                self.normalized_strings[index],
                self.raw_strings[index],
            )

            if score < self.min_score:
                continue

            result = dict(self.items[index])   # copy original item
            result['score'] = round(score, 2)
            result['score_pct'] = round(score, 2)
            results.append(result)

        # Sort by score descending, return top_k
        results.sort(key=lambda x: x['score'], reverse=True)
        return results[:top_k]

    def search_with_field_scores(self, query: str, top_k: int = 20) -> List[Dict[str, Any]]:
        """
        Same as search() but also returns per-field scores.
        Useful for bucketing results into High / Medium / Low priority.
        """
        results = self.search(query, top_k)
        query_n = normalize(query)

        for r in results:
            r['field_scores'] = {
                field: blend_score(
                    query_n,
                    normalize(str(r.get(field, ''))),
                    str(r.get(field, '')).lower(),
                )
                for field in self.text_fields
            }

        return results
```

# ── Example 1: Basic product search ──────────────────────────────────────────

products = [
    {"id": 1, "name": "China Hookah Small",        "brand": "",        "category": "Hookahs"},
    {"id": 2, "name": "Gold Color Grinder 4 Part", "brand": "DnK",     "category": "Grinders"},
    {"id": 3, "name": "10 Inch Glass Beaker 9MM",  "brand": "",        "category": "Glass Pipes"},
    {"id": 4, "name": "5 Hour Energy Berry Box",   "brand": "5 Hour",  "category": "Energy Drinks"},
    {"id": 5, "name": "Marlboro Red 20s",           "brand": "Marlboro","category": "Cigarettes"},
    {"id": 6, "name": "JUUL Pod Virginia 5%",       "brand": "JUUL",    "category": "Vape"},
]

engine = FuzzySearchEngine(products)

# Exact match
print(engine.search("hookah"))
# → [{"id":1, "name":"China Hookah Small", "score":100.0, ...}]

# Typo
print(engine.search("hooka"))
# → [{"id":1, "name":"China Hookah Small", "score":~88.0, ...}]

# Wrong word order
print(engine.search("4 part grinder gold"))
# → [{"id":2, "name":"Gold Color Grinder 4 Part", "score":100.0, ...}]

# Partial keyword
print(engine.search("glass"))
# → [{"id":3, "name":"10 Inch Glass Beaker 9MM", "score":~85.0, ...}]

# Brand search
print(engine.search("marlboro"))
# → [{"id":5, "name":"Marlboro Red 20s", "score":100.0, ...}]
```

# ── Example 3: Bucket into High / Medium / Low ────────────────────────────────

def bucket(results, high=70, medium=50, low=35):
    buckets = {"high": [], "medium": [], "low": []}
    for r in results:
        max_field = max(r.get('field_scores', {}).values(), default=0)
        if   max_field >= high:   buckets["high"].append(r)
        elif max_field >= medium: buckets["medium"].append(r)
        elif max_field >= low:    buckets["low"].append(r)
    return buckets

results = engine.search_with_field_scores("energy drink", top_k=10)
grouped = bucket(results)
print("HIGH:",   [r['name'] for r in grouped['high']])
print("MEDIUM:", [r['name'] for r in grouped['medium']])
print("LOW:",    [r['name'] for r in grouped['low']])
```

## Score Interpretation

| Score | Meaning | Show to user as |
|---|---|---|
| 90–100 | Near-perfect match | ✅ High Match (green) |
| 70–89 | Strong match | ✅ High Match (green) |
| 50–69 | Good match | 🔵 Good Match (blue) |
| 35–49 | Possible match | 🟡 Possible Match (yellow) |
| < 35 | Irrelevant | ❌ Discard |

## Key Points for Your Other Project

1. **No model download needed** — RapidFuzz is pure Python, works offline
2. **No pre-computation** — scores are calculated at query time, no index to build
3. **`_build_index()`** just pre-computes the normalized strings once at startup — O(n) time
4. **Each search query** runs in O(n) time against all items — fast enough for up to ~50,000 items
5. **The two-pass approach** (fast WRatio scan → full blend re-score) is the performance optimization — skip it if you have < 5,000 items and just run the full blend directly
6. **`text_fields`** is configurable — point it at whatever fields make sense for your data
